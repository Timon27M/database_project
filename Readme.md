# Vehicle Database

Проект базы данных для хранения и анализа информации о транспортных средствах.

## Описание

База данных содержит информацию об автомобилях, мотоциклах и велосипедах, включая их характеристики и производителей.

## Структура базы данных

- **Vehicle** — общая информация о транспортных средствах (производитель, модель, тип)
- **Car** — характеристики автомобилей (объём двигателя, мощность, цена, трансмиссия)
- **Motorcycle** — характеристики мотоциклов (объём двигателя, мощность, цена, тип)
- **Bicycle** — характеристики велосипедов (количество передач, цена, тип)

## Реализованные задачи

1. Поиск спортивных мотоциклов с мощностью более 150 л.с. и ценой менее $20 000
2. Выборка автомобилей, мотоциклов и велосипедов по заданным критериям с объединением результатов

## Запуск

1. Выполнить скрипт `initial_tables.sql` для создания таблиц
2. Выполнить скрипт `add_data_to_tables.sql` для наполнения данными
3. Запустить запросы из `task_1.sql` и `task_2.sql`


# Car Racing Database

База данных для хранения информации об автомобильных классах, машинах и результатах гонок.

## Важно: В задаче №5 раздела Car Racing Database опечатка в задании low_position_count у car_name = Audi A4 равен 1.

## Описание

Проект содержит данные об автомобилях, их классах и результатах участия в гонках. Позволяет анализировать средние позиции автомобилей и классов.

## Структура базы данных

- **Classes** — классы автомобилей (тип, страна, количество дверей, объём двигателя, вес)
- **Cars** — автомобили (название, класс, год выпуска)
- **Races** — гонки (название, дата)
- **Results** — результаты (автомобиль, гонка, занятая позиция)

## Реализованные задачи

1. Поиск автомобилей с наименьшей средней позицией в каждом классе
2. Определение автомобиля с наилучшей средней позицией среди всех
3. Поиск классов с наименьшей средней позицией и всех их автомобилей
4. Выборка автомобилей со средней позицией лучше средней по классу
5. Определение классов с наибольшим количеством автомобилей с низкой средней позицией (более 3.0)

## Запуск

1. Выполнить скрипт `initial_tables.sql` для создания таблиц
2. Выполнить скрипт `add_data_to_tables.sql` для наполнения данными
3. Запустить запросы из `task_1.sql` — `task_5.sql`


# Hotel Booking Database

База данных для управления бронированиями отелей и анализа предпочтений клиентов.

## Описание

Проект содержит информацию об отелях, номерах, клиентах и бронированиях. Позволяет анализировать статистику бронирований и предпочтения клиентов.

## Структура базы данных

- **Hotel** — отели (название, расположение)
- **Room** — номера (тип, цена, вместимость, привязка к отелю)
- **Customer** — клиенты (имя, email, телефон)
- **Booking** — бронирования (даты заезда/выезда, привязка к номеру и клиенту)

## Реализованные задачи

1. Поиск клиентов с более чем двумя бронированиями в разных отелях, подсчёт среднего времени пребывания
2. Анализ клиентов с более чем двумя бронированиями в разных отелях и суммой трат более $500
3. Категоризация отелей по средней стоимости номера и определение предпочтений клиентов

## Запуск

1. Выполнить скрипт `initial_tables.sql` для создания таблиц

# Employee Management Database

База данных для управления сотрудниками, проектами и задачами в организации.

## Описание

Проект содержит информацию об отделах, сотрудниках, их ролях, проектах и задачах. Позволяет анализировать иерархию подчинения и загрузку сотрудников.

## Структура базы данных

- **Departments** — отделы компании
- **Roles** — должности сотрудников
- **Employees** — сотрудники (имя, должность, руководитель, отдел, роль)
- **Projects** — проекты (название, даты, отдел)
- **Tasks** — задачи (название, исполнитель, проект)

## Реализованные задачи

1. Рекурсивный поиск всех подчинённых Ивана Иванова с информацией о проектах и задачах
2. Рекурсивный поиск подчинённых с подсчётом количества задач и прямых подчинённых
3. Поиск менеджеров с подчинёнными и подсчётом общего количества подчинённых на всех уровнях

## Запуск

1. Выполнить скрипт `initial_tables.sql` для создания таблиц
2. Выполнить скрипт `add_data_to_tables.sql` для наполнения данными
3. Запустить запросы из `task_1.sql` — `task_3.sql`
